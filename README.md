# BingX Trading Scanner – Техническая документация


---
Этот репозиторий содержит два Python-модуля для **сканирования фьючерсного рынка** биржи BingX. Система отслеживает открытый интерес (OI), ставки финансирования и технические индикаторы (EMA + MACD) для генерации торговых сигналов в реальном времени.

## 📦 Модуль 1: `bingx_client.py`

### Класс `BingxClient`

**Назначение**: Низкоуровневое взаимодействие с REST API BingX (как публичные, так и авторизованные запросы).

#### Инициализация
```python
client = BingxClient(api_key="", api_secret="", testnet=True)
```
- `api_key`, `api_secret` – могут быть пустыми для публичных эндпоинтов.
- `testnet` – если `True`, используется `https://open-api-vst.bingx.com`, иначе основная сеть.

#### Публичные методы (без ключей)

| Метод | Описание | Возвращает |
|-------|----------|------------|
| `get_all_tikers()` | Возвращает список всех фьючерсных символов (например, `["BTC-USDT", "ETH-USDT"]`). | `List[str]` |
| `get_klines(symbol, interval, limit)` | OHLCV данные в виде pandas DataFrame. Колонки: `time, open, high, low, close, volume`. | `pd.DataFrame` |
| `get_open_insterest(symbol)` | Текущий открытый интерес в USDT. | `float` или `None` |
| `get_premium_index(symbol=None)` | Возвращает маркировочную цену, индексную цену и последнюю ставку финансирования для одного или всех символов. | `List[Dict]` или `Dict` |
| `get_mark_price(symbol)` | Маркировочная цена (используется для ликвидаций). | `float` |
| `get_funding_rate(symbol)` | Последняя ставка финансирования (8‑часовая). | `float` |
| `get_index_price(symbol)` | Цена базового индекса. | `float` |

#### Приватные методы (требуют API ключи)

| Метод | Описание |
|-------|----------|
| `get_positions()` | Возвращает все открытые позиции. |
| `place_market_order(side, qty, symbol, stop, tp, pos_side_BOTH, reduceOnly)` | Рыночный ордер с опциональными SL/TP. |
| `set_sl(symbol, qty, side, stop, one_way_mode)` | Установка только стоп-лосса. |
| `set_multiple_tp(symbol, qty, mark_price, side, tp_levels, both)` | Установка нескольких тейк-профитов (делит количество). |
| `set_trailing(symbol, side, qty, activation_price, priceRate, BOTH)` | Активация трейлинг-стопа. |
| `set_leverage_bx(symbol, side, leverage, one_way_mode)` | Установка плеча для стороны позиции. |
| `cancel_existing_orders(symbol)` | Отмена всех открытых ордеров по символу. |
| `get_open_orders(symbol)` | Возвращает список открытых ордеров. |
| `is_sl_exists(symbol)` | Проверяет, существует ли стоп-лосс ордер. |

#### Вспомогательные методы
- `_to_bingx_symbol(symbol)` – преобразует `"BTCUSDT"` → `"BTC-USDT"`.
- `_public_request(path, params)` – GET запрос без подписи.
- `_request(method, path, params)` – подписанный запрос.
- `get_server_time_offset()` – компенсация разницы времени сервера и локального.

## 📡 Модуль 2: `bingx_scanner.py`

### Описание
Непрерывно сканирует все фьючерсные символы BingX каждые `SCAN_INTERVAL_SECONDS`. Применяет фильтры и технический анализ, выводит сигналы в консоль.

### Конфигурационные константы (в начале файла)

| Константа | Значение по умолчанию | Описание |
|-----------|----------------------|----------|
| `SCAN_INTERVAL_SECONDS` | 60 | Пауза между циклами сканирования (сек). |
| `TIMEFRAME` | `"1h"` | Таймфрейм свечей для индикаторов (`15m`, `1h`, `4h`). |
| `KLINES_LIMIT` | 150 | Количество свечей для расчёта EMA/MACD. |
| `OI_THRESHOLD` | 1_000_000 | Минимальный OI в USDT (абсолютный фильтр). |
| `OI_CHANGE_PERCENT` | 10.0 | Минимальное изменение OI в % (если `USE_OI_CHANGE=True`). |
| `FUNDING_RATE_THRESHOLD` | 0.0005 (0.05%) | Минимальная абсолютная ставка финансирования для прохода фильтра. |
| `EMA_FAST` | 9 | Период быстрой EMA. |
| `EMA_SLOW` | 21 | Период медленной EMA. |
| `MACD_FAST` | 12 | Быстрый период MACD. |
| `MACD_SLOW` | 26 | Медленный период MACD. |
| `MACD_SIGNAL` | 9 | Период сигнальной линии MACD. |
| `USE_OI_CHANGE` | `False` | `True` = использовать % изменения, `False` = абсолютный OI. |
| `OI_CHANGE_WINDOW` | 24 | Количество свечей для вычисления изменения OI (если `USE_OI_CHANGE=True`). |
| `FILTER_MODE` | `"AND"` | `"AND"` = оба фильтра (ставка+OI), `"OR"` = хотя бы один. |

### Основные функции

#### `get_timeframe_minutes(timeframe)`
Преобразует `"1h"` → 60, `"15m"` → 15 и т.д.

#### `class OIHistoryTracker`
Хранит снимки OI в памяти для вычисления процентного изменения за скользящее окно.

- `add_entry(symbol, oi)` – добавляет временную метку + OI, удаляет старые записи.
- `get_percentage_change(symbol, current_oi)` – возвращает `(change_percent, status_message)`.

#### `calculate_indicators(df)`
Возвращает pandas Series: `(ema_fast, ema_slow, macd_histogram)`.

#### `check_technical_signals(df)`
Анализирует последние 4 закрытые свечи на предмет:

- пересечения EMA (быстрая пересекает медленную)
- пересечения гистограммой MACD нулевой линии
- учёт задержки в 1 свечу для снижения шума

Возвращает `dict` с ключами:
- `signal`: `"BUY"`, `"SELL"` или `"NEUTRAL"`
- `ema_fast_val`, `ema_slow_val`, `macd_hist_val`
- `details`: описание пересечений

**Логика сигнала**:  
Оба индикатора (EMA и MACD) должны быть согласованы (бычий/медвежий) И должно произойти хотя бы одно пересечение (EMA или MACD) за последние 1–2 закрытые свечи.

#### `main(client)`
Бесконечный цикл:
1. Получить все тикеры → множество символов.
2. Получить премиум-индексы для всех (ставка финансирования, марк-цена).
3. Отфильтровать по `FUNDING_RATE_THRESHOLD`.
4. Отфильтровать по OI (абсолютному или процентному изменению).
5. Для каждого кандидата загрузить свечи → рассчитать технические сигналы.
6. Вывести форматированный сигнал, если `BUY` или `SELL`.

**Контроль частоты запросов**: Пауза 0.1–0.2 сек между вызовами API для избежания ограничений.

### Пример вывода
```
==================================================
[2025-01-15 14:30:00] СИГНАЛ: BUY (LONG) для ETH-USDT
   Цена маркировки: 3200.50
   Open Interest: 2,345,678.90 USDT (абсолютный порог)
   Funding Rate: +0.0620% (8h)
   EMA(9) > EMA(21): 3200.12 / > / 3195.43
   MACD гистограмма: 12.45 (пересечение EMA вверх и пересечение MACD вверх)
==================================================
```

## 🚀 Использование

1. **Укажите API ключи** (для публичного сканирования они могут быть пустыми):
   ```python
   api_key = "your_key"
   api_secret = "your_secret"
   client = BingxClient(api_key, api_secret, testnet=False)
   ```

2. **Настройте параметры сканера** в `bingx_scanner.py` (пороги, таймфреймы, режим фильтрации).

3. **Запустите**:
   ```bash
   python bingx_scanner.py
   ```

4. **Остановка**: `Ctrl+C`.

## ⚙️ Технические примечания

- **История OI** хранится в памяти (не сохраняется). При первом запуске расчёт изменения OI может показывать «накапливается история», пока не наберётся достаточно измерений.
- **Преобразование символов**: внутренний формат BingX использует дефис (например, `BTC-USDT`). Клиент автоматически конвертирует из `BTCUSDT`.
- **Порог ставки финансирования** использует абсолютное значение: учитываются как сильно положительные, так и сильно отрицательные ставки (высокий funding указывает на дисбаланс).
- **Задержка сигнала**: сканер использует закрытые свечи (индексы -2 и -3) для избежания перерисовки (repainting).
- **Синхронизация времени**: `get_server_time_offset()` используется для подписанных запросов во избежание ошибок timestamp.
